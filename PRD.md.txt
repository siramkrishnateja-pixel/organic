🥛 Organic Dairy & Farm Products Platform
Product Requirements Document (PRD)
[Updated: April 2026 — DB & Payment decisions locked]
________________________________________
1. 📌 Product Overview
A subscription-first e-commerce platform for:
•	Organic dairy products (milk, ghee, curd, paneer)
•	Organic vegetables & farm produce
•	Organic oils and other natural products
•	Future: grocery add-ons
Inspiration:
•	Catalog-driven platforms (like Indira Foods)
•	Fresh delivery models (like Little India)
Key Differentiators:
•	Subscription-first model (core)
•	Organic trust & transparency
•	Integrated operations (inventory + production + delivery)
•	Admin-driven business intelligence (P&L, analytics)
________________________________________
2. 🎯 Goals
•	Enable daily/recurring delivery of milk & organic products
•	Reduce waste via demand prediction
•	Build trust through organic transparency
•	Provide complete admin control over operations & finance
________________________________________
3. 👥 User Roles
3.1 Customer (User)

3.2 Admin

•	Analyze revenue, costs, and profit (P&L dashboard)

10. 🧱 TECH STACK (CONFIRMED)

•	REST API (using Supabase Connect API / PostgREST)
________________________________________
Database ✅ DECIDED
•	Provider: Supabase (PostgreSQL)
•	Connection Method: Supabase Connect API (PostgREST auto-generated REST)
•	All DB access via Data Access Layer (DAL) — no direct DB calls from UI
•	DAL talks to Supabase via supabase-js → PostgREST REST API → PostgreSQL
________________________________________
Authentication ✅ DECIDED
•	Provider: Supabase Auth
•	Method: Phone OTP (primary), Email (optional)
________________________________________
Storage ✅ DECIDED
•	Provider: Supabase Storage (S3-compatible)
________________________________________
Payments
•	Phase 1 (Testing): Dummy Payment Gateway (no real money, simulated flow)
•	Phase 2 (Production): Real gateway — TBD (Razorpay recommended for India/UPI)
________________________________________
Notifications
•	Phase 1: SMS OTP via Supabase+Twilio; In-app via notifications table; Email via Resend
•	Phase 2: Push notifications (FCM), WhatsApp (Twilio)
________________________________________
11. 🧠 DATA ACCESS STRATEGY (DAL)
All database interactions go through a Data Access Layer (DAL):
Example Functions:
•	createUser()
•	getOrdersByDateRange()
•	updateInventory()
•	calculatePnL()
Benefits:
•	Easy database switching
•	Clean separation of logic
•	Scalable architecture
